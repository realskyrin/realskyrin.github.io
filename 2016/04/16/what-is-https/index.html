HTTPS 及 SSL

发表于 2016-04-16 | 更新于 2018-12-05 | 评论数：

HTTP VS HTTPS

超文本传输​​协议安全（HTTPS）是HTTP的安全版本，它是在您的浏览器和您连接的网站之间发送数据的协议。HTTPS结尾处的’S’代表’安全’。这意味着浏览器和网站之间的所有通信都是加密的。HTTPS通常用于保护网上银行和在线购物订单等高度机密的在线交易。Web浏览器（如Internet Explorer，Firefox和Chrome）还会在地址栏中显示挂锁图标，以便直观地表明HTTPS连接有效。


HTTPS 如何工作？

HTTPS页面通常使用两种安全协议之一来加密通信 - SSL（安全套接字层）或TLS（传输层安全）。 TLS和SSL协议都使用所谓的“不对称”公钥基础设施（PKI）系统。非对称系统使用两个“密钥”来加密通信，“公共”密钥和“私人”密钥。任何用公钥加密的东西只能用私钥解密，反之亦然。 正如名字所暗示的那样，“私钥”应该受到严格的保护，并且只能被私钥的所有者访问。在网站的情况下，私钥仍保留在Web服务器上。相反，公钥旨在分发给任何需要能够解密用私钥加密的信息的人和每个人。

什么是 HTTPS 证书？

当您向网页请求HTTPS连接时，网站最初会将其SSL证书发送至您的浏览器。此证书包含开始安全会话所需的公钥。基于这个初始交换，您的浏览器和网站会启动’SSL握手’。 SSL握手涉及共享秘密的生成，以建立您和网站之间的独特安全连接。 在HTTPS连接期间使用可信SSL数字证书时，用户将在浏览器地址栏中看到挂锁图标。当在网站上安装扩展验证证书时，地址栏会变绿。

为什么需要 SSL 证书？

所有通过常规HTTP连接发送的通信都是“纯文本”，并且可以被任何黑客读取，这些黑客设法闯入浏览器和网站之间的连接。如果“沟通”在订单上，并且包含您的信用卡信息或社会安全号码，则会带来明显的危险。通过HTTPS连接，所有通信都被安全加密。这意味着即使有人设法进入连接，他们也无法解密您和网站之间传递的任何数据。

SSL 协议的握手过程

根据上图得出 SSL 协议握手分五个步骤

第一步，Visitor 给出协议版本号、一个客户端生成的随机数（Client random），以及客户端支持的加密方法。
第二步， Cloud 确认双方使用的加密方法，并给出数字证书、以及一个服务器生成的随机数（Server random）。
第三步，Visitor 确认数字证书有效，然后生成一个新的随机数（Premaster secret），并使用数字证书中的公钥，加密这个随机数，发给 Cloud 。
第四步， Cloud 使用自己的私钥，获取Visitor 发来的随机数（即Premaster secret）。
第五步，Visitor 和 Cloud 根据约定的加密方法，使用前面的三个随机数，生成”对话密钥”（session key），用来加密接下来的整个对话过程。

超文本传输​​协议安全的好处

HTTPS证书的主要优点是：

  • 客户信息（如信用卡号码）已加密且无法截取
  • 访问者可以验证您是注册商家并且拥有该域名
  • 客户更可能信任并完成使用HTTPS的网站购买
